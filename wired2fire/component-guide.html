Page Scoping

How to apply page-specific styles without conflicts

Implementation Pattern

To keep styles organized and prevent conflicts between pages, use page-scoped CSS with a class on the body element:

<!-- In HTML file -->
<body class="gaming-pcs-page">
  <!-- Page content -->
</body>

/* In w2f.css */
/* GAMING PCS PAGE STYLES */
.gaming-pcs-page .category-hero {
  /* Styles only apply to gaming-pcs-page */
}

.gaming-pcs-page .product-card {
  /* Styles only apply to gaming-pcs-page */
}

.gaming-pcs-page .trust-section {
  /* Styles only apply to gaming-pcs-page */
}

Existing Page Scopes

Page Body Class CSS Section
Gaming PCs .gaming-pcs-page /* GAMING PCS PAGE STYLES */
Configurator .configurator-page /* CONFIGURATOR PAGE STYLES */
Homepage (none - uses global styles) /* HOMEPAGE STYLES */

Adding a New Page

1. Add page class to body tag:
   <body class="flight-sim-page">

2. Create section in w2f.css:
   /* FLIGHT SIM PAGE STYLES */
   
   .flight-sim-page .hero-section {
     /* Page-specific styles */
   }
   
   .flight-sim-page .product-grid {
     /* Page-specific styles */
   }

3. Reuse global components when possible
4. Only override when necessary for page-specific design

Best Practices

  • Use page scoping for page-specific styles to prevent conflicts
  • Reuse global components like buttons, accordions, tabs when possible
  • Keep inline styles minimal - only for unique one-off values like background images
  • Follow naming conventions - Use descriptive class names like .product-card, .trust-section
  • Group related styles - Keep all accordion styles together, all button styles together
  • Comment your sections - Use /* SECTION NAME */ headers for clarity
  • Test responsiveness - Check mobile, tablet, desktop breakpoints
